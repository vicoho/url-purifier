# 贡献指南

感谢您对网址净化工具的兴趣！我们欢迎所有形式的贡献。

## 如何贡献

### 报告问题

如果您发现了 bug 或有功能建议，请通过 [Issue](https://github.com/vicoho/url-purifier/issues) 提交。

提交 Issue 时，请尽可能提供以下信息：
- 问题的简要描述
- 复现步骤
- 期望行为 vs 实际行为
- 浏览器版本和操作系统
- 相关截图（如有）

### 提交代码

1. Fork 本仓库
2. 创建您的特性分支：`git checkout -b feature/AmazingFeature`
3. 提交更改：`git commit -m 'feat: add some amazing feature'`
4. 推送到分支：`git push origin feature/AmazingFeature`
5. 创建 Pull Request

### 提交规范

本项目使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

- `feat:` 新功能
- `fix:` 修复 bug
- `docs:` 文档更新
- `style:` 代码格式（不影响功能）
- `refactor:` 代码重构
- `perf:` 性能优化
- `test:` 测试相关
- `chore:` 构建/工具相关

### 代码风格

- 使用 2 空格缩进
- 使用单引号
- 添加适当的注释
- 保持函数简洁（单一职责）

## 开发环境

本项目是纯前端项目，无需任何构建工具：

```bash
# 直接在浏览器中打开 index.html 即可
cd url-purifier
open index.html
```
