## 1. 产品概述

### 1.1 产品背景
在社交媒体、论坛、评论区等场景中，用户分享网址时常常会在URL中穿插文字、表情符号或特殊字符，以规避平台的自动屏蔽机制。例如：

- `https://exa😊mple.com/abc`
- `https://exa mple.com`
- `https://exa-mple.com（请去掉横线）`
- `https://exa...mple.com`

这些被"污染"的网址无法直接点击访问，需要手动清理，体验极差。

### 1.2 产品定位
一款轻量、高效的在线网址净化工具，帮助用户一键还原被穿插文字/表情的原始网址，使其恢复可访问状态。

### 1.3 目标用户
- 社交媒体活跃用户
- 论坛/社区浏览者
- 需要频繁处理分享链接的内容工作者

## 2. 核心功能

### 2.1 网址净化（核心功能）

#### 功能描述
自动识别输入文本中的被污染网址，去除穿插其中的多余字符，还原为标准的可访问URL。

#### 支持的污染类型

| 污染类型 | 示例 | 净化结果 |
|---------|------|---------|
| 表情符号穿插 | `https://exa😊mple.com` | `https://example.com` |
| 空格穿插 | `https://exa mple.com` | `https://example.com` |
| 中文字符穿插 | `https://exa测mple.com` | `https://example.com` |
| 多余标点穿插 | `https://exa...mple.com` | `https://example.com` |
| 提示性文字包裹 | `https://example.com（请复制到浏览器）` | `https://example.com` |
| 换行/制表符穿插 | `https://exa\nmple.com` | `https://example.com` |
| 多种污染混合 | `https://exa😊 mple...com` | `https://example.com` |

#### 净化规则

1. **URL结构保留**：保留协议头（`http://` / `https://`）、域名、路径、查询参数、锚点等标准URL结构
2. **非法字符移除**：移除URL主体中的表情符号、中文字符、多余空格、换行符、制表符
3. **保留必要符号**：保留URL中合法的符号（如 `-`、`_`、`/`、`?`、`=`、`&` 等）
4. **智能边界识别**：准确识别URL的起止位置，避免误删URL前后的正常文本
5. **多URL处理**：支持同时识别并净化文本中的多个网址

### 2.2 批量处理

- 支持在输入框中粘贴包含多个被污染网址的长文本
- 自动识别文本中所有符合URL模式的字符串
- 分别净化每个网址，保留非URL文本不变

### 2.3 结果展示

- 清晰展示净化后的网址列表
- 每个网址可一键复制
- 提供"全部复制"功能
- 支持直接点击访问（在新标签页打开）

### 2.4 历史记录

- 本地存储最近20条净化记录
- 每条记录包含：原始文本、净化后的URL列表、处理时间
- 历史记录列表直接显示净化后的URL链接（可点击跳转）
- 支持查看详情（显示原始文本，可一键复制）
- 支持清空历史记录
- 与净化结果以Tab形式切换展示

## 3. 用户交互流程

### 3.1 主流程

```
┌─────────────────┐
│   进入网页      │
└────────┬────────┘
         ▼
┌─────────────────┐
│  粘贴污染文本   │
│  （输入框）     │
└────────┬────────┘
         ▼
┌─────────────────┐
│  点击净化按钮   │
│  / 自动触发     │
└────────┬────────┘
         ▼
┌─────────────────┐
│  Tab切换        │
│ [净化结果|历史] │
└────────┬────────┘
         ▼
┌─────────────────┐
│  复制/访问网址  │
└─────────────────┘
```

### 3.2 交互细节

#### 输入阶段
- 提供大文本输入框，支持多行粘贴
- 支持 `Ctrl+V` 直接粘贴触发自动净化（可选）
- 显示输入字符数统计

#### 处理阶段
- 点击"净化"按钮后显示处理中状态（加载动画）
- 处理时间应 < 500ms（普通文本）
- 净化成功后自动切换到「净化结果」Tab

#### 输出阶段（Tab形式展示）

**Tab栏**：
- 「净化结果」Tab：展示本次净化后的网址列表
- 「历史记录」Tab：展示本地存储的历史净化记录
- 首次进入页面时，默认展示「历史记录」Tab（因为尚无净化结果）
- 用户执行净化操作后，自动切换到「净化结果」Tab

**净化结果Tab内容**：
- 以卡片/列表形式展示净化后的每个网址
- 每个网址项包含：
  - 净化后的完整URL（可点击）
  - "复制"按钮
  - "访问"按钮（外部链接图标）
- 提供"复制全部"按钮
- 如果未检测到网址，显示友好提示

**历史记录Tab内容**：
- 以时间倒序展示历史记录列表
- 每条记录包含：
  - 处理时间戳
  - 净化后的URL链接列表（可直接点击跳转，超出宽度显示省略号）
  - 净化出的URL数量
  - "查看详情"按钮（弹窗显示原始文本，支持一键复制）
  - "删除"按钮（删除单条记录）
- 提供"清空全部"按钮（带二次确认）
- 空状态时显示提示"暂无历史记录，快去净化第一个网址吧"

## 4. 界面设计规范

### 4.1 整体风格（Claude风格）

- **设计理念**：Claude风格——温暖、优雅、专业、可信赖。采用暖橙色调为主色，营造亲切友好的使用氛围。界面简洁但不冷淡，细节处体现精致感。
- **色调**：浅色模式为主（Claude标志性的暖白背景），提供深色模式切换
- **布局**：居中单栏布局，核心操作区域突出，留白充足，呼吸感强
- **视觉特征**：
  - 圆角卡片（border-radius: 12px ~ 16px）
  - 柔和的阴影（而非生硬边框）
  - 精致的图标（Lucide风格或类似）
  - 温暖的渐变点缀

#### Tab设计说明

- Tab栏使用细线条分隔，避免厚重感
- 激活Tab下方有橙色指示条（2px），与主色调呼应
- Tab切换有平滑过渡动画
- 净化结果Tab和历史记录Tab内容区高度自适应

### 4.5 动效规范

- **按钮悬停**：轻微上移（translateY -2px）+ 阴影加深，过渡 200ms（Claude风格偏好位移而非缩放）
- **结果出现**：淡入 + 上滑动画，stagger 间隔 80ms
- **Tab切换**：内容区淡入淡出（opacity + translateX），过渡 250ms
- **复制成功**：按钮文字变为"已复制" + 橙色背景闪烁，2秒后恢复
- **加载状态**：按钮内显示旋转动画
- **卡片悬停**：轻微阴影加深 + 边框色变暖，过渡 150ms

## 5. 技术方案

### 5.1 技术栈

- **框架**：纯原生 HTML + CSS + JavaScript（零依赖，单文件）
- **无需构建工具**：直接保存为 `.html` 文件即可运行
- **兼容性**：现代浏览器（Chrome、Firefox、Safari、Edge 最新两版）

### 5.2 核心算法

#### URL识别
使用正则表达式匹配文本中的URL模式：

```javascript
// 基础URL匹配正则
const urlRegex = /https?:\/\/[^\s\u4e00-\u9fa5]+/gi;

// 更精确的正则（需考虑被污染的情况）
const urlPattern = /https?:\/\/[\w\-._~:/?#[\]@!$&'()*+,;=]+/i;
```

#### 净化算法

```javascript
function purifyURL(dirtyUrl) {
  // 1. 移除表情符号（Unicode emoji范围）
  // 2. 移除中文字符
  // 3. 移除多余空格、换行、制表符
  // 4. 移除URL中不应该出现的标点符号
  // 5. 保留URL合法字符
  return cleanedUrl;
}
```

#### 多URL提取策略

1. 先通过正则找出所有可能的URL片段
2. 对每个片段应用净化算法
3. 验证净化结果是否为合法URL（通过URL构造函数）
4. 返回有效URL列表

### 5.3 本地存储

- 使用 `localStorage` 存储用户主题偏好
- 使用 `localStorage` 存储最近20条净化记录
- 历史记录数据结构：

```javascript
{
  id: "uuid",
  timestamp: 1717584000000,
  originalText: "原始文本内容...",
  urls: [
    "https://example.com",
    "https://another.com"
  ]
}
```

- 历史记录按时间倒序排列
- 超过20条时自动删除最旧的记录

### 5.4 文件结构

```
url-purifier/
└── index.html          # 单文件应用（HTML+CSS+JS内联）
```

## 6. 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 输入为空 | 显示提示"请输入包含网址的文本" |
| 未检测到网址 | 显示提示"未检测到网址，请检查输入" |
| 净化后URL不合法 | 标记为"疑似网址"，仍展示但添加警告样式 |
| 输入文本极长（>10万字） | 显示进度提示，分批处理 |
| 用户粘贴图片/文件 | 忽略非文本内容，提示"请粘贴文本内容" |
| 浏览器不支持 | 显示基础版本提示，建议升级浏览器 |

## 7. 非功能需求

### 7.1 性能
- 页面加载时间 < 1s
- 净化处理时间 < 500ms（普通长度文本）
- 支持同时处理50+个URL

### 7.2 可访问性
- 支持键盘操作（Tab导航、Enter触发）
- 按钮和链接有清晰的焦点样式
- 颜色对比度符合WCAG AA标准

### 7.3 安全
- 纯前端处理，不发送任何数据到服务器
- 用户隐私100%本地保护
- 外部链接打开时添加 `rel="noopener noreferrer"`
